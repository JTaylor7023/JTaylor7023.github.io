Javascript Fundamentals

Javascript as drama!

Think of HTML as a script of a play. It defines what is on stage, the actors, their lines and a plays structure.

CSS is what goes on behind the scenes. The costumes, the lightning and background scenery. It makes things look good.

Javascript is the Director (or God). It brings things to life. It tells actors when to enter or exit and how to react or change mood or behaviour.

It controls how things behave.

Control Flow and Loops

Control Flow is like playing a video game.

         
          startLevel();

You have a key-you open a chest.

         if (player.hasKey) {
          openTreasureChest();
      }

You fall in a pit and lose a life.

      if (player.fellInPit) {
        loseLife();
      }

You defeat the boss

        
          defeatBoss()

The level ends.

        
          endLevel()

This is control flow deciding what happens in a situation.

Loops

Now you fighting hordes of zombies...they keep coming and coming.

  • While there are enemies:
  • Attack zombies
  • Dodge zombies
  • Repeat
        
          while (enemies.length > 0) {
            let enemy = enemies.pop();
            attack(enemy);
            dodge(enemy.attack);
          }

How about a "for" loop for collecting coins.

        
          for (let i = 0; i < coins.length; i++) {
            collect(coins[i]);
          }

Basically:

  • Control Flow: If you do this than that happens
  • Loops: "Keep doing this until...."

Arrays

Arrays are like a row of numbered mailboxes.

They are ordered lists where each box has a number (index).

      
        let fruits = ["apple", "banana", "cherry"];
console.log(fruits[0]); // "apple"
  • You access items by number(starting at 0)
  • Think: "Give me the first fruit" → fruits[0]

Use arrays when order matters or you're using a list of similar items

Oblects

An object is an unordered collecton of key value pairs-like folders with labels.

      
        let person = {
          name: "Alex",
          age: 30,
          city: "New York"
        };
        
        console.log(person["name"]); // "Alex"
        console.log(person.age);     // 30

Functions.

You can think of a function as a mini program inside your code

You can give it stuff to do (parameters) and it rreturns a ressult.

Lets say you make pancakes every morning, rather than writing out the recipe everytime....

      
        function makePancakes(eggs, flour, milk) {
          mix(eggs, flour, milk);
          cook();
          return "Pancakes are ready!";
        }

Then you just say(call)

      
        makePancakes(2, "1 cup", "1/2 cup");
